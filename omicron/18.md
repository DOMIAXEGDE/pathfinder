Define the least nonnegative residue of an integer z modulo a positive integer M as the unique integer r such that 0 <= r < M and r is congruent to z modulo M. By default, reduce each coordinate id modulo that component’s domain size M_c. Since the deterministic encoder already yields 0 <= id_c < M_c, the default least-residue coordinate is the id itself. 16.cpp must also implement a deterministic integer-seed system. This seed system is not a pseudorandom generator. It is an exact rank/unrank addressing mechanism over the finite tensor-row universe determined by the current alphabets, length policies, and selected instance count.

For a fixed 17.txt configuration, let the six ordered components be:

p, q, m, g, alpha, beta.

For each component c, let M_c be the raw string-domain size produced by that component’s finite alphabet and length policy. For fixed length L and alphabet size r, M_c = r^L. For variable length [L_min, L_max], M_c = sum from ell = L_min to L_max of r^ell.

Define the raw tensor-row universe U as:

U = D_p × D_q × D_m × D_g × D_alpha × D_beta,

where D_c = {0, 1, ..., M_c - 1}. Its size is:

R = M_p * M_q * M_m * M_g * M_alpha * M_beta.

Every raw six-coordinate tensor row has a unique row ordinal J in {0, ..., R - 1}. Use big-endian mixed-radix ranking:

J = (((((id_p * M_q + id_q) * M_m + id_m) * M_g + id_g) * M_alpha + id_alpha) * M_beta + id_beta).

The inverse unranking must recover the component ids by repeated quotient and remainder in reverse component order:

id_beta = J mod M_beta,
J = floor(J / M_beta),
id_alpha = J mod M_alpha,
J = floor(J / M_alpha),
id_g = J mod M_g,
J = floor(J / M_g),
id_m = J mod M_m,
J = floor(J / M_m),
id_q = J mod M_q,
J = floor(J / M_q),
id_p = J.

For each recovered raw component id, reconstruct the canonical input string using the inverse of the ordinal text encoder. For a fixed-length component, write the id in base r using exactly L digits, then map each digit back to the corresponding alphabet symbol. For a variable-length component, first find the unique length L such that offset(L) <= id < offset(L) + r^L, where offset(L) = sum from ell = L_min to L - 1 of r^ell. Then decode id - offset(L) as a fixed-length base-r ordinal of length L.

Let N be the user-specified basis row count. By default, a basis tensor dataset is an ordered N-tuple of raw tensor rows, with repetition allowed. Therefore the total number of ordered basis tensor datasets under the current configuration is:

S = R^N.

Call S the basis-address-space size.

A basis tensor dataset B = [row_0, row_1, ..., row_{N-1}] has a unique basis address K in {0, ..., S - 1}. Use big-endian base-R ranking:

K = (((row_0 * R + row_1) * R + row_2) ... * R + row_{N-1}).

The inverse unranking must recover rows by repeated quotient and remainder in reverse row order:

row_{N-1} = K mod R,
K = floor(K / R),

and so on until row_0 is recovered.
